## Executive Summary

The site has strong local-service SEO potential, but there are several **critical technical SEO problems** that should be fixed before deeper content optimization.

The biggest issues are:

- **Sitemap is broken/incomplete** — currently only 17 location pages are listed.
- **Canonical/domain mismatch** — code uses both `sprinkleranddrains.com` and `texasbestsprinklers.com`.
- **Homepage is a client component** — hurts performance and limits ideal metadata/schema behavior.
- **Conflicting LocalBusiness schema** — phone number, hours, address, and `@id` are inconsistent.
- **Many service pages only have title metadata** — missing descriptions, canonical URLs, OG/Twitter metadata.
- **Global scripts are too heavy** — Google Maps, Smartlook, performance observers, preloading helpers, and manual prefetch logic run globally.
- **Image strategy is inconsistent** — duplicate hero preloads, raw `<img>` usage, local PNG/JPEG assets, and hardcoded global LCP preload regardless of route.

If fixed, the site would be in much better shape for local SEO, crawlability, Core Web Vitals, and Google Business/local pack support.

## 2. Canonical domain mismatch

**Severity:** Critical  
**Files:**

- [src/app/page.tsx](cci:7://file:///Users/christopherhancock/Desktop/dev/repos/texas-best-sprinklers/src/app/page.tsx:0:0-0:0)
- [src/components/BlogPreview.tsx](cci:7://file:///Users/christopherhancock/Desktop/dev/repos/texas-best-sprinklers/src/components/BlogPreview.tsx:0:0-0:0)
- [src/app/layout.tsx](cci:7://file:///Users/christopherhancock/Desktop/dev/repos/texas-best-sprinklers/src/app/layout.tsx:0:0-0:0)
- [next-sitemap.config.js](cci:7://file:///Users/christopherhancock/Desktop/dev/repos/texas-best-sprinklers/next-sitemap.config.js:0:0-0:0)

The official domain appears to be:

```txt
https://sprinkleranddrains.com
```

But the homepage uses this fallback:

```tsx
const baseUrl = typeof window !== 'undefined' ? window.location.origin : 'https://texasbestsprinklers.com';
```

And [BlogPreview](cci:1://file:///Users/christopherhancock/Desktop/dev/repos/texas-best-sprinklers/src/components/BlogPreview.tsx:22:0-124:2) uses:

```tsx
"@id": `https://texasbestsprinklers.com/blog/${post.slug}`
```

This creates mixed signals across schema, canonicals, and sitemaps.

### Why this matters

Google may treat these as separate entities/domains, causing:

- Diluted entity authority
- Duplicate content/canonical confusion
- Bad structured-data entity merging
- Incorrect rich result references

### Recommended Fix

Add a single canonical site URL to `SITE_CONFIG`:

```ts
siteUrl: 'https://sprinkleranddrains.com'
```

Then replace all hardcoded domain references with that.
